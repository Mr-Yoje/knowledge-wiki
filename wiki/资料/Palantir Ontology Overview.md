# Palantir Ontology Overview

## 来源信息

- **来源**: Palantir Foundry 官方文档
- **URL**: https://palantir.com/docs/foundry/ontology/overview/

## 一句话摘要

Palantir Ontology 是组织的运营层，将数字资产与现实世界实体连接，作为企业的数字孪生。

## 解决的问题

企业数据分散在不同系统中（数据集、虚拟表、模型），缺乏统一语义层。传统数据编目和 schema 设计不能支撑端到端的用户工作流。

## 关键结构

1. **Object types（对象类型）和 Link types（链接类型）**：定义语义
2. **Action types（动作类型）和 Functions（函数）**：定义动力学（kinetics）——实现变更并遵守治理
3. **Interfaces（接口）**：对象类型的多态抽象
4. **Ontology-aware applications**：Object Explorer、Workshop、Vertex 等应用深度集成

## 可沉淀知识点

- Ontology 包含两层：语义元素（对象、属性、链接）和动能元素（动作、函数、动态安全）
- 接口提供了对象类型的多态能力，允许以一致的方式与共享共同形状的对象类型交互
- Ontology 的目标是赋能组织中大规模、更好的决策
